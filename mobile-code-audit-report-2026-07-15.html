一、架构概览

移动端采用独立多入口设计，与 PC 端在构建层面完全分离，但共享底层数据层（Pinia Store / Service / Supabase）。

三、问题与 Bug 清单

🔴 P0 — 严重问题

P0M-01首页点击任务无法查看/编辑详情

首页点击任务卡片跳转到 app-tasks 并带 ?id=xxx，但 MobileTaskApp 的 onMounted 拿到 id 后直接 router.replace 清空了 query，没有打开任何详情（源码注释也写明 "for now just clear the query"）。移动端完全没有任务详情/编辑页面，用户无法查看描述、修改时间、优先级、删除任务。

MobileTaskApp.vue:110-117 · MobileHome.vue:102-104

建议：实现任务详情底部弹层（Sheet）或详情页，支持编辑标题/日期/时间/优先级/删除。这是移动端任务管理的核心缺失。

🟠 P1 — 功能缺陷

P1M-02重复的 auth.init() 调用

会话初始化在三处重复执行：路由守卫 beforeEach、MobileApp.vue onMounted、以及 App 挂载。虽然 auth store 有 initialized 幂等标志不会重复请求，但逻辑分散，且 MobileApp 的 onMounted 与路由守卫存在竞态可能。

mobile/router/index.ts:50 · MobileApp.vue:36

建议：统一由路由守卫负责会话恢复，MobileApp onMounted 仅做数据 load。
P1M-03退出登录逻辑重复实现两份

MobileApp.vue handleLogout() 与 MobileProfile.vue handleLogout() 是几乎完全相同的 15 行清理逻辑（flush/cleanup/broadcast/reload）。MobileApp 通过 defineExpose 暴露但实际调用的是 Profile 内部的副本，前者形同死代码。

MobileApp.vue:69-84 · MobileProfile.vue:74-90

建议：抽取为 composables/useLogout.ts 或 service 函数，两端复用，删除重复。
P1M-04任务搜索按钮为装饰，无实际功能

MobileTaskApp 导航栏的搜索图标没有绑定任何点击事件；MobileApps 顶部搜索框标记为 readonly，纯装饰。用户点击无反应，产生功能预期落差。

MobileTaskApp.vue:130-135 · MobileApps.vue:62

建议：短期移除或置灰这两个入口；长期实现搜索。
P1M-05待办 Tab 缺少"延期/已完成"筛选逻辑

待办列表 filteredTodos 只处理了 today 筛选，当用户切到"延期"或"已完成"chip 时，待办列表不做任何过滤（TodoItem 无 completed 概念，但 UI 仍显示这些 chip，行为不一致）。

MobileTaskApp.vue:72-78

建议：待办 Tab 下隐藏"延期/已完成"chip，或明确禁用。
P1M-06快速添加任务无成功反馈

FAB 快速添加任务后仅关闭弹层，无 Toast 提示、无滚动定位到新任务。若添加的任务不在当前筛选条件下（如非今日），列表无变化，用户会误以为添加失败。

MobileTaskApp.vue:100-108

建议：添加成功后 Toast 提示，并在需要时切换筛选到能看见新任务的视图。
P1M-07主题切换仅支持 3 档，与 PC 端 5 档不一致

Profile 的 cycleTheme 仅在 light/dark/auto 间循环，而 ThemeMode 类型定义了 5 档（含 zuru、tencent）。移动端用户无法使用 ZURU / 腾讯蓝品牌主题。此外主题 key 手动写死 'cleannotes_theme'，与 useTheme 内部持久化可能重复/不一致。

MobileProfile.vue:62-68 · useTheme.ts:3

建议：改为主题选择弹层，或至少纳入全部可用主题；持久化交给 useTheme 统一管理。

Profile 菜单的数据同步状态是写死的绿色"已同步"文本，不反映真实同步状态（无 pending/失败/离线态）。误导用户。

MobileApps 底部提示"长按图标可拖拽排序"，但代码没有任何拖拽排序实现，属虚假提示。

MobileApps.vue:134-136

P2M-12硬编码 hex 颜色，违反项目 CSS 变量公约

多处 .vue style 直接写死 hex（如 #FF3B30 #FF9500 #34C759 #C7C7CC），项目约定禁止在 .vue style 出现 hex、应用 CSS 变量。这些颜色在暗黑模式下不会自适应。

MobileHome.vue:92-96 / :317-327 · MobileTaskApp.vue:407 等

P2M-13页面切换过渡动画可能残留（transition mode）

MobileLayout 用 <transition name="page" mode="out-in">，但对应的 .page-* 过渡样式需确认已在 style.css 定义，否则动画无效或产生闪烁。

MobileLayout.vue:9

prod 目录存在源码中不存在的构建产物（如 MobileCalendar、TaskDetailSheet），系历史构建残留（robocopy 非 /MIR 不清理）。可能造成部署内容与当前代码认知偏差。

四、结论与优先级建议

  • 核心闭环缺失（最高优先）：M-01 任务详情/编辑功能缺失，导致移动端只能"看"和"勾选"，无法真正管理任务。建议优先补齐详情弹层。
  • 功能诚实性：M-04/M-10/M-11 存在多处"看起来能用但实际是装饰"的 UI（搜索、同步状态、拖拽排序），应移除或补实现，避免误导。
  • 代码整洁：M-02/M-03/M-08/M-09 属重复代码与死代码，建议一次性抽取复用并清理。
  • 设计系统一致性：M-07/M-12 主题档位与颜色硬编码，需对齐 PC 端 CSS 变量体系与主题能力，尤其保证暗黑模式适配。
  • 整体评价：移动端骨架（入口分离、路由、鉴权、数据共享、PWA）已搭建完整且清晰，登录与今日任务浏览可用；但业务功能仍处早期，12 个应用入口中仅任务部分可用，距完整可用尚有明显差距。
