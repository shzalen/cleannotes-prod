# 2026-07-09 工作日志

## 修复：Mermaid 语法错误显示位置（2026-07-09 08:20）

### 问题
- 备忘录中使用 Mermaid 图表时，语法错误信息会泄漏到整个页面底部显示
- 根因：`mermaid.default.render(id, code)` 在解析失败时会向 `document.body` 追加错误 SVG/DOM 元素，且原代码仅用 `finally` 清理 `d${id}` 幽灵元素，未清理 mermaid 创建的其他错误 DOM

### 修复
- `src/components/MermaidNodeView.vue` `renderMermaid()` 函数：
  - 将 `mermaid.default.render()` 包裹在 `try/finally` 中
  - `finally` 块中除清理 `d${id}` 幽灵元素外，额外用 `querySelectorAll` 清理 `#mermaid-error`、`.mermaid-error`、`[id^="mermaid-error"]` 等 mermaid 创建的错误 DOM
  - 错误信息通过 `refreshDiagram()` 的 catch 捕获，显示在节点内部的 `.rte-mermaid-error` 区域（含错误图标 + 消息 +「编辑修复」按钮）
- CSS：`.rte-mermaid-error` 添加 `margin: 12px` + `min-height: 80px` + `justify-content: center`，确保错误显示在节点内居中
