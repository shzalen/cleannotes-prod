## 回到顶部按钮功能（最终修复版）
- **根因**：各视图根元素均有 `overflow-y: auto; height: 100%`，自身成为滚动容器，`.app-main` 实际不滚动。之前绑定到 `.app-main` 的 scroll 监听永远不触发
- **最终方案**：`document.addEventListener('scroll', handler, true)` 捕获阶段监听所有元素的 scroll 事件
- 关键知识点：scroll 事件不冒泡，但可在捕获阶段拦截；`e.target` 获取实际滚动元素
- 记录 `lastScrolledEl` 用于点击时回滚到正确的容器
- 仅在 App.vue 放一个 BackToTop 实例，无需各视图单独引入

## 登录页面配色统一（与系统主题对齐）
- 问题：登录页全部使用硬编码绿色系 hex 值（#3b9a5c、#1a472a、#5cb878 等），与系统 CSS 变量体系不一致，且不支持暗黑模式
- 第一轮修复（错误）：用 --color-primary（蓝靛色 #4f6cf7）替换，导致登录页变蓝，与系统内部绿色基调更不协调
- 第二轮修正（正确）：经排查 AppSidebar/HomeView/TasksView 实际配色，确认系统主强调色是 --color-success（绿色 #22c55e）而非 --color-primary（蓝靛色）
- 最终配色方案：
  - 背景渐变：var(--color-bg-1) → var(--color-success-light) → var(--color-bg-2)（中性灰白底+微妙绿调）
  - 品牌 icon：var(--color-success-light) bg + var(--color-success-text) color（与侧栏品牌 icon 一致）
  - 按钮：var(--color-success) / hover:var(--color-success-text) / disabled:color-mix(success 35%, border)
  - 装饰圆/聚焦环/验证码区：success 色系 color-mix 半透明
  - 文字/边框/卡片：系统标准 text/border/surface 变量
  - 暗黑模式自动适配（CSS 变量在 [data-theme="dark"] 下自动切换）
- style.css 新增变量：--color-primary-hover / --color-primary-disabled（保留，可用于其他场景）
- 零 hex 遗留，构建通过
