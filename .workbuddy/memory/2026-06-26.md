## 弹窗面板滚动翻转问题（已完成）

### 问题
用户反馈：当页面内容有滚动时，`/` 面板弹出被遮盖，需要将面板显示在上方。

### 根因
面板永远固定在光标下方（`coords.bottom + 4`），当下方空间不足时会被视口裁剪。

### 修复方案
在空间不足时自动翻转面板到光标上方显示。

### 改动文件

#### `src/extensions/SlashCommand.ts`
- `SlashState` 接口新增 `flipUp?: boolean` 可选字段

#### `src/components/RichTextEditor.vue`
- 新增 `computeSlashCoords(coords)` 函数：
  - 估算面板高度 320px，判断下方/上方空间
  - 下方空间足够 → 显示在下方（默认）
  - 下方不足但上方足够 → 翻转到上方
  - 都不够 → 选空间较大的一侧
- `watch(editor.value?.state)` 中面板刚打开时调用 `computeSlashCoords` 计算初始位置
- `onEditorScroll` 中滚动时重新调用 `computeSlashCoords`
- 面板渲染后 `nextTick` 中读取实际高度并纠正位置

#### `src/extensions/MemoMention.ts`
- `startRepaintLoop()` 中的定时刷新函数加入空间判断：
  - 读取面板实际高度（已知时）或估算值 220px
  - 下方空间不足时翻转到上方
  - 加入边界检查，防止面板超出视口

## 修复斜杠命令 `/` 无响应

### 问题
用户反馈：修复后输入 `/` 没有任何反应，斜杠命令面板不弹出。

### 根因
1. `SlashCommand.ts` 的 `handleTextInput` 消费文本输入事件时，只更新了插件状态，没有把 `/` 和后续查询字符实际写入文档，导致编辑器无可见反馈，且 `deleteRange` 无内容可删。
2. `RichTextEditor.vue` 通过 `watch(() => editor.value?.state)` 同步插件状态到 Vue，Vue 无法可靠追踪 ProseMirror state 变化，面板经常不响应。

### 修复方案
1. `handleTextInput` 现在会原子地执行 `insertText` 并更新 `slashCommand` 插件状态：
   - 在块首输入 `/` 时，把 `/` 插入文档并打开面板。
   - 面板已打开时，把输入字符追加到文档并更新 `query` / `range`。
2. `handleKeyDown` 中 Backspace 逻辑完善：
   - `query === ''` 时关闭面板，让 ProseMirror 删除 `/`。
   - `query` 非空时删除查询末尾字符并同步插件状态。
3. `RichTextEditor.vue` 移除脆弱的 `watch(editor.value?.state)`，改用 Tiptap `onTransaction` 回调同步插件状态，确保每次 transaction 后 Vue 状态及时更新。

### 改动文件
- `src/extensions/SlashCommand.ts`
- `src/components/RichTextEditor.vue`
