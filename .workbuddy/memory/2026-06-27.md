## 修复：表格操作控件点击后无法插入/删除行列
- **现象**：用户在 IIS 中刷新后，点击表格操作菜单里的「插入/删除行列」无反应
- **根因**：Tiptap 的 `ChainedCommands` 是不可变对象，`tableAction()` 中先 `const chain = editor.chain().focus()`，之后 `chain.addRowBefore()` 返回的是新对象，但末尾执行的是原始 `chain.run()`，所以只执行了 `focus()`，真正的表格命令根本没跑
- **修复**：把 `const chain` 改成 `let chain`，每个 switch case 重新赋值：`chain = chain.addRowBefore()`，最后 `chain.run()` 执行完整命令链
- 构建通过 ✅（新产物 RichTextEditor-BqN4So-8.js 21.60KB）

## 二次修复：表格操作控件仍无法操作行列

  3. 删除相关命令（deleteRow/Column/Table）也统一走同一条 chain，避免提前 return 导致的状态不一致

## 三次修复：表格操作控件报错 `Cannot read properties of undefined (reading 'domAtPos')`
- **现象**：用户刷新后出现 `Uncaught TypeError: Cannot read properties of undefined (reading 'domAtPos')`，onSelectionUpdate 触发报错
- **根因**：`updateTableControls()` 中 `const { state } = editor.value` 用了 `.value`，但后面 `editor.view.domAtPos(...)` 漏了 `.value`，`editor` 是 Vue ref 对象，没有 `view` 属性
- **修复**：改为 `editor.value.view.domAtPos(...)`
## 四次修复：表格操作控件仍无法操作行列
- **现象**：用户反馈刷新后仍然无法插入/删除行列，无报错
- **根因**：Tiptap chain 的 `createChainableState` 在创建时会缓存 `selection`，虽然 `setTextSelection` 会修改 transaction 的 selection，但 chain 中后续表格命令拿到的 `state.selection` 仍是旧值，导致 `addRowBefore` 等命令无法定位当前单元格
- **修复**：`tableAction()` 不再使用 chain，而是分步执行：
  1. `editor.value.commands.setTextSelection(savedTableSelection.value)` 立即 dispatch 恢复选区
  2. `editor.value.commands.focus()` 聚焦编辑器
  3. 单独调用 `editor.value.commands.addRowBefore()` 等表格命令，每个命令都会读取最新的 editor state
  4. 失败时在控制台输出 `console.warn('[RichTextEditor] tableAction failed:', action)` 便于排查
- 构建通过 ✅（新产物 RichTextEditor-DTJ6rJLP.js 21.90KB）

## 七次修复：表格控件完全不生效（按钮未显示 + 命令时机问题）
