## 修复 H5 底部空白：100dvh→position:fixed（11:40）
- 根因：`#app { height: 100dvh }` 在部分 iOS 版本上 `100dvh` 略小于实际视口高度，导致底部露出白色空隙
- 修复：`#app` 改为 `position: fixed; top: 0; left: 0; right: 0; bottom: 0`，不依赖 dvh 单位，强制铺满整个视口
- 这是比 `safe-bottom` 调负值更正确的方案（safe-bottom 为系统安全区高度，不应篡改）

## TabBar 底部留白根治方案（11:38）
- 根因：`align-items: stretch` 导致 `.tabbar__item` 被安全区 padding 拉伸，图标随之上移；`max(var(--safe-bottom), 12px)` 兜底不足，底部未覆盖完全
- 修复：`.tabbar` 增加 `align-items: flex-start`，`.tabbar__item` 改为固定 `height: var(--tabbar-height)`，确保图标/标签只在 52px 内容区内居中，不随安全区高度变化
- 将底部安全区兜底从 12px 提高到 16px，覆盖底部留白/手势条区域
- `.m-layout__body` 和 `.home-fab` 的 bottom/padding-bottom 同步使用 `max(var(--safe-bottom), 16px)`，保持与 TabBar 一致
