## TabBar 底部空白彻底修复 — padding-bottom + content-box 方案（14:35）

## TabBar 彻底重构 — App Shell Flex 布局（17:46）
- **结论**：之前的 padding-bottom、content-box、safe-area 等修修补补都无法根本解决 TabBar 贴底问题，因为根因是 **高度链断裂** — `MobileApp.vue` 中 `<router-view>` 没有显式高度，导致 `MobileLayout` 的 `height: 100%` 解析为内容高度而非视口高度
- **全面重构 6 个文件**：
  - `MobileApp.vue`：模板改为 `.app-shell`（fixed flex column）> `.app-content`（flex:1）> router-view + `MobileTabBar`（flex-shrink:0），TabBar 回归文档流底部
  - `MobileTabBar.vue`：移除 `position: fixed; bottom: 0; left: 0; right: 0`，改为 `flex-shrink: 0` 在 flex 容器中自然定位
  - `MobileLayout.vue`：`height: 100%` → `flex: 1; min-height: 0`（直接参与父 flex 布局，不再依赖高度链）；删除 `padding-bottom: var(--tabbar-height)`
  - `MobileLogin.vue`：`min-height: 100%` → `flex: 1; min-height: 0; overflow-y: auto`（同样修复高度链依赖）
  - `MobileCalendar.vue`：`.cal-content` padding-bottom 80px → 16px（TabBar 回归流后不再需要大留白）
  - `MobileProfile.vue`：`.profile-content` padding-bottom 80px → 16px（同上）
- 架构优势：TabBar 不再依赖 `position:fixed`（移动端浏览器 notorious bug 源头），完全在正常 flex 流中，高度链清晰可靠

## 首页/日历页橡皮筋滚动效果恢复（17:47）
- **根因**：前次 TabBar 重构时，`.home-page` 和 `.cal-page` 保留了 `overflow: hidden`（用于防止滚动穿透），但它同时裁剪了内层滚动容器（`.home-pull-refresh` / `.cal-pull-refresh`）的橡皮筋回弹视觉
- **修复**：移除 `.home-page` 和 `.cal-page` 的 `overflow: hidden`（只保留注释说明）
- `.m-layout__body` 层已有 `overflow: hidden` 防止穿透，页面级冗余
- 影响文件：`MobileHome.vue`、`MobileCalendar.vue`
