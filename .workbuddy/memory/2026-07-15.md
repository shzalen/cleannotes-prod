## 触控防误触优化 (useTouchInteraction)
- 问题：首页下拉时容易误触任务项（touch 位移极小被判定为 tap 打开详情）

- 新增 `src/mobile/composables/useTouchInteraction.ts`
  - 判定规则：位移 > 15px → 取消；位移 ≤ 15px + duration < 1s → tap；位移 ≤ 15px + duration ≥ 1s → long-press（带振动）
  - 泛型设计 `useTouchInteraction<T>`，返回 `{ handleTouchStart, handleTouchMove, handleTouchEnd }`
- MobileHome.vue 和 MobileCalendar.vue 任务项触控已改为使用 composable

   - 内容区包裹 `<van-pull-refresh>`，`v-model="refreshing"` + `@refresh` → `taskStore.load(true)`
   - `.home-content` 改为 `min-height: 100%` + `touch-action: pan-y`（滚动由 van-pull-refresh 容器管理）
   - 新增 `.home-pull-refresh { flex:1; min-height:0; overflow-y:auto; }`

### 天气定位持久化（MobileWeatherWidget.vue）
- **根因**：`maximumAge: 600000` 只是浏览器内存缓存，PWA 关闭即丢失
- **修复**：
  - 成功定位后保存坐标到 `localStorage`（key: `cleannotes_weather_coords`，含 lat/lon/ts）
  - 挂载时优先读取 localStorage 缓存，10 分钟内有效 → 直接请求天气，不触发定位权限弹窗
  - 缓存过期或不存在才调用 `navigator.geolocation.getCurrentPosition()`
  - 新增 `locationFailed` ref，定位失败时显示圆形刷新按钮（SVG 重试箭头图标）
  - 点击按钮 → `requestLocation()` 重新请求定位
- 移除旧 `geoWatchId` / `onUnmounted` 清理（改用 getCurrentPosition 一次性获取）

## 天气弹窗"今天"数据与首页不一致修复 (18:28)
- 问题：弹窗 7 天预报"今天"显示"暴雨"，但首页天气显示"阴天"
- 根因：首页用 Open-Meteo `current` 端点（实时观测 weather_code），弹窗用 `daily` 端点（日预报模型 weather_code）——两个独立数据源，daily 预报可能预测当天晚些时候有暴雨但当前实测是阴天
- 修复：弹窗模板中"今天"卡片直接使用 `weather.value`（与首页共享的实时观测数据），后续 6 天仍用 `dailyForecast.filter(d => !d.isToday)` 的 daily 预报数据
- 构建：2865 modules, 24.87s；robocopy 同步到生产目录
