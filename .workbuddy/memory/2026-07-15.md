## 触控防误触优化 (useTouchInteraction)
- 问题：首页下拉时容易误触任务项（touch 位移极小被判定为 tap 打开详情）
- 方案一：位移阈值拦截（> 15px 视为滚动/下拉，取消一切交互）
- 新增 `src/mobile/composables/useTouchInteraction.ts`
  - 判定规则：位移 > 15px → 取消；位移 ≤ 15px + duration < 1s → tap；位移 ≤ 15px + duration ≥ 1s → long-press（带振动）
  - 泛型设计 `useTouchInteraction<T>`，返回 `{ handleTouchStart, handleTouchMove, handleTouchEnd }`
- MobileHome.vue 和 MobileCalendar.vue 任务项触控已改为使用 composable
- 构建成功：2862 modules, 28s；robocopy 同步 132 文件到生产目录

## 橡皮筋阻尼效果恢复
- 问题：首页/日历页橡皮筋阻尼效果不如"我的"页面自然
- 根因：van-pull-refresh 内部嵌套 overflow-y:auto 容器，两层滚动冲突，van-pull-refresh 无法感知滚动边界
- 修复：移除首页和日历页的 van-pull-refresh，改用原生 overflow-y:auto + -webkit-overflow-scrolling:touch 滚动容器
- 同时移除未使用的 showToast 导入
- 构建：2862 modules, 24.75s；robocopy 同步 131 文件到生产目录

## Flexbox 滚动修复
- 问题：移除 van-pull-refresh 后首页和日历页完全无法滚动
- 根因：flexbox 中 `flex: 1` 子元素默认 `min-height: auto`，即使设了 `overflow-y: auto` 也不会缩小；外层容器缺少 `overflow: hidden` 约束
- 修复：`.home-page` / `.cal-page` 加 `overflow: hidden`；`.home-content` / `.cal-content` 加 `min-height: 0`
- 构建：2862 modules, 30.27s；robocopy 同步 133 文件到生产目录
