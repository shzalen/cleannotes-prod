## iOS PWA 状态栏 bug 修复（09:57）
- **问题**：iOS PWA `apple-mobile-web-app-status-bar-style=black-translucent` 在 app 从后台切回时失效（iOS 已知行为），状态栏从透明变黑色不透明 → 出现"黑条 + 白缝 + 蓝头"
- **白缝根因**：原 `.home-header` 内部用 `<div class="home-header__safe-area" />` 空 div 撑出 safe-top 高度（无背景色），状态栏变黑时该区域显示 body 白色背景
- **修复**：
  1. `mobile.html` meta 改为 `default`（状态栏始终是 iOS 系统白色，**两次切回效果完全一致**，最稳定）
  2. 移除 `home-header__safe-area` 空 div，改用 `.home-header { padding-top: var(--safe-top) }`（header 蓝色背景延伸到屏幕最顶部覆盖状态栏区域）
- **iOS PWA 限制**：web 不可在 runtime 控制状态栏颜色，meta 是唯一手段且仅首次启动生效；"状态栏也是主题色"的真正沉浸式需用 webview 容器（如 Capacitor）打包
- **构建**：2837 模块转换成功；prod 已同步
