## 2026-07-13 GitHub Pages 部署问题排查

- 用户反馈 GitHub Pages（https://shzalen.github.io/cleannotes-prod/）下 AI 设置保存报错
- 初判为 Web Crypto 安全上下文问题，但用户确认 HTTPS + 独立标签页，环境正常
- 用户部署最新版本后控制台仍报错：
  1. `ReferenceError: toLocalDate is not defined`（HomeView 组件）
  2. `[AI] Failed to encrypt API key: SyntaxError: Unexpected end of JSON input`
- 根因分析：
  - `TaskHeatmap.vue` 使用了 `toLocalDate()` 但没有从 `@/utils/time` 导入，被 Vite 打包进 HomeView chunk 后运行时未定义
  - `supabase.ts` 的 `request()` 直接调用 `res.json()`，当代理/PostgREST 返回空 body 时（非 204）会抛出 JSON parse 错误；该错误被 `ai.ts` catch 误标为"加密失败"
- 修复：
  - `TaskHeatmap.vue` 增加 `import { toLocalDate } from '@/utils/time'`
  - `supabase.ts` 的 `request()` 改为 `res.text()` + `JSON.parse()`，空 body 时返回 `null`
  - `ai.ts` 的 `persistConfig()` 错误文案区分：Web Crypto 不可用 / 存储网络错误 / 真正加密失败
- 构建：vite build exit 0（22.50s），robocopy 同步到 prod
- 主仓库 commit (57bc7f8)：TaskHeatmap import + empty-body Supabase response fix
- prod 仓库 commit (dist sync)
