# 2026-07-10

## 修复：周报 AI 总结"AI不可用"问题

**根因**：`App.vue` 的 `onMounted` 中加载了 `taskStore.load()` 和 `growthStore.load()`，但缺少 `aiStore.load()`。`aiStore.config` 初始为空 `{ apiUrl: '', apiKey: '', model: '' }`，只有当某个组件（SettingsView / AiView / AiChat）调用 `aiStore.load()` 后才会从 storage 加载真实配置。

周报生成时 `callAiForSummary()` 直接读取 `aiStore.config`，如果尚无组件触发过 `load()`，config 为空字符串，判定"AI 不可用"。

**修复**（两处）：
1. `App.vue`：在 `onMounted` 中添加 `aiStore.load()`，确保应用启动时 AI 配置即加载
2. `weeklyReport.ts` 的 `callAiForSummary()`：调用前加 `await aiStore.load()` 作为防御性兜底

已构建并部署到 prod。

## 事故：robocopy /MIR 误删 prod 目录 .git

`robocopy /MIR` 镜像模式会删除目标中源不存在的文件，包括 prod 目录的 `.git` 仓库。已重新 `git init` + `git remote add origin` 恢复。
**此后部署禁止用 `/MIR`**，改用 `/E /XD .git`（排除 .git 目录）。

## 修复：IIS 子目录部署资源 404

**根因**：`vite.config.ts` 的 `base` 为 `/`，构建产物使用绝对路径（如 `/assets/index-xxx.js`）。prod 部署在 IIS 子目录 `v1.3.0` 下，浏览器请求 `/assets/...` 而非 `/v1.3.0/assets/...`，导致 404。

**修复**：将 `vite.config.ts` 的 `base` 从 `'/'` 改为 `'./'`，构建产物使用相对路径。router 使用 `createWebHashHistory`，不受 base 影响。已重新构建并部署到 prod。
