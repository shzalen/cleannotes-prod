### 首页今日任务数据逻辑（对齐 PC TodayProgress.vue）
- 筛选：`startDate===today` 或 `startDate<today且未完成` 或 `!startDate回退createdAt` 或 `completedAt===today`
- 排序：未完成在前（startDate+startTime升序），已完成在后（completedAt降序）
- 交互：点击状态圆圈 → `requestToggleStatus`（已完成需确认对话框）；点击卡片 → 跳转子应用

### 移动端自动重定向（设备自适应）
- `main.ts`：PC 应用挂载前检测 `isMobileDevice() && !isForcePC()` → `window.location.replace('./mobile.html')`，Vue 应用不挂载
- `mobile.ts`：桌面端误访问 `mobile.html` → `window.location.replace('./index.html')`
- `router/index.ts` 路由守卫兜底：移动端访问 PC 路由 → `window.location.replace('./mobile.html')`（原指向旧 `/h5/tasks`，已改）
- 旧 H5 路由（`/h5/*`）保留向后兼容，但移动端不再跳转过去
- `force_pc` localStorage flag 仍有效：用户可强制桌面版

### 移动端样式修复：CSS 变量缺失
- **根因**：`mobile/style.css` 只有 `@import "tailwindcss"` + 基础样式，缺少全部 CSS 变量定义（`--color-bg-1`、`--color-primary` 等）。这些变量在 PC 端 `style.css` 中定义，但移动端是独立入口不加载 PC 样式
- **修复**：将 PC 端 `style.css` 的 4 套主题变量（light/dark/zuru/tencent）完整复制到 `mobile/style.css`
- **注意**：共享组件（如 `ConfirmDialog.vue`）也依赖这些变量，必须包含完整变量集（`--color-danger-text`、`--color-warning-text` 等）
- 教训：独立入口的样式表必须自包含所有 CSS 变量定义，不能假设共享组件会自带
