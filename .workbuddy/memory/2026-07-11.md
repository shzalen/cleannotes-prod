# 2026-07-11

## 全面安全与性能审计（第二轮）

对整个系统执行了 5 维度并行审计（认证授权 / 数据存储传输 / 前端安全 / 构建加载 / 运行时网络），发现 37 项问题 + 18 项已通过检查。

### 严重 (3 项)
- **S/P-01 跨标签页同步缺失**：todo.ts / weeklyReport.ts / growth.ts 从不调用 broadcastChange()，跨标签页数据不同步
- **P-01 memo 搜索 DOMParser**：stripHtml() 对每个 memo 调用 DOMParser，搜索 100 条 = 100 次解析，极卡
- **P-02 dragover 监听器泄漏**：RichTextEditor add/removeEventListener 传入不同函数引用，监听器永不移除

### 高 (8 项)
- S-02 CSP 'unsafe-eval' 削弱 XSS 防护
- S-03 Link URL 验证仅阻止 javascript: 前缀，可被 data:/vbscript:/空格等绕过
- S-04 登出 fire-and-forget，缓存 JWT 未及时清除
- S-05 AI API Key 明文存储在数据库
- S-06 旧 RLS 策略可能仍使用 x-user-id header
- S-07 Supabase anon key 硬编码 + 项目未使用环境变量
- S-08 DiagView 暴露全部 localStorage 含 JWT
- P-03 @supabase/supabase-js (8.2MB) 被拉入入口 chunk

### 关键修复建议
1. todo/weeklyReport/growth 添加 broadcastChange() — 改动量小收益高
2. stripHtml 改用正则 replace 替代 DOMParser
3. dragover 监听器提取具名函数引用
4. Link validate 改为 URL 白名单（http/https/mailto/tel）
5. logout() 先 setCachedAccessToken(null) 再后台 signOut
6. vite.config.ts 添加 vendor-supabase manualChunk
