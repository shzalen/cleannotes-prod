## 修复 Supabase Health Check 401 错误
- **问题**：`hybrid.ts` 的 `checkSupabaseHealth()` HEAD 请求每15秒发一次到 `/rest/v1/` 根路径，PostgREST 不支持对根路径的 HEAD 请求，持续返回 401

- **修复**：改为 `GET /rest/v1/cleannote_tasks?limit=0`，使用实际表名 + GET 方法，正确通过 PostgREST 认证；同时保留 Authorization 头

## 修复备忘录图片/文件上传失败 — base64 内嵌

### 问题
- 图片和文件上传依赖 Supabase Storage bucket `cleannote_attachments`，但 bucket 需要手动在 Dashboard 创建
- 用 anon key + RLS 策略（依赖 `auth.uid()`）上传可能因无真实 JWT 用户而失败

### 修复方案

- 图片：FileReader 读取为 base64 data URL → `setImage({ src: dataUrl })` 直接嵌入
- 文件：FileReader 读取为 base64 data URL → `insertContent(<a href="data:..." download>)` 嵌入下载链接
- Image extension `allowBase64: true`
- 大小限制：5MB，超限弹 alert
- 文件类型校验：图片仅允许 `image/*`
- 保留 supabase.ts 中 Storage 函数但编辑器不再引用

### 优缺点
- 优点：零外部依赖，即插即用，同步完全自包含
- 缺点：base64 增大内容体积约 33%，但个人笔记场景可接受

## 修复文件上传后卡片/编辑器中无法打开（DOMPurify data: URL 拦截）

### 根因
DOMPurify 默认只允许 `http/https/mailto/...` 协议在 `href` 中，`data:` URL 被清空。
文件链接 `<a href="data:...">` sanitize 后只剩纯文本文件名，无法点击下载。

### 修复方案
1. 文件上传生成的 HTML 从 `<a href="data:...">` 改为 `<span class="rte-file-link" data-file="base64..." data-filename="xxx">`
2. DOMPurify ALLOWED_ATTR 新增 `data-file`、`data-filename`（ALLOWED_TAGS 中 `a` → `span`）
3. RichTextEditor：`onMounted` 在 `rteWrapperRef` 上监听 click → `.rte-file-link` → `triggerFileDownload()`
4. MemoView：`onMounted` 在 `document` 上监听 click → `.rte-file-link` → 同样 Blob 下载逻辑
5. `triggerFileDownload()`：base64 解码 → Uint8Array → Blob → Object URL → 动态 `<a>` download → revoke

### 影响范围
- RichTextEditor.vue：handleFileUpload 生成格式变更 + handleAttachmentClick 重构为 triggerFileDownload
- MemoView.vue：导入 onBeforeUnmount + document click 代理 + DOMPurify 白名单 + CSS 样式
- 编辑器内和卡片视图中的文件附件均可点击下载

## 修复备忘录文件上传 — Tiptap 剥离 data-* 属性导致无法下载

### 根因
`insertContent('<span data-file="base64...">')` 插入后，Tiptap 的 HTML 解析器（ProseMirror DOMParser）会剥离不认识的 `data-*` 属性。保存后的 HTML 只剩 `<span class="rte-file-link">📎 filename (size)</span>`，`data-file` 属性丢失，点击下载的 handler 找不到 base64 数据。

### 修复方案
创建自定义 Tiptap Node 扩展 `src/extensions/FileAttachmentExtension.ts`：
- `group: inline`, `atom: true`, `selectable: true` — 内联原子节点，不可内联编辑
- `parseHTML`: 识别 `<span class="rte-file-link" data-file data-filename>` 并提取属性 + textContent 作为 displayText
- `renderHTML`: 输出 `<span class="rte-file-link" data-file data-filename>displayText</span>`
- `addCommands`: `insertFileAttachment({ file, filename, displayText })` 替代 `insertContent` 原始 HTML

### 改动文件
- **新建** `src/extensions/FileAttachmentExtension.ts`
- **RichTextEditor.vue**: 注册 FileAttachment 扩展；`handleFileUpload` 改用 `insertFileAttachment` 命令；CSS `.rte-file-attachment .rte-file-link` → `.rte-file-link`（无外层包裹）

### 构建
- `npx vite build` 成功，零错误
