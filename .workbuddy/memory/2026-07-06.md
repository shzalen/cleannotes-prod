## GitHub Pages 部署配置
- 项目路由已是 `createWebHashHistory()`，无需改动
- `vite.config.ts` `base` 默认 `/`（IIS 本地），通过 `--base` CLI 参数切换
- `package.json` 新增 `build:gh-pages`：`vite build --base /cleannotes/`
- `.github/workflows/deploy.yml`：push main 时自动 `npm ci` → `npm run build:gh-pages` → peaceiris/actions-gh-pages 部署到 gh-pages 分支
- 用户需在 GitHub 仓库 Settings → Pages 中将 Source 设为 gh-pages 分支 `/ (root)`
- 手动部署方案：`scripts/deploy-gh-pages.sh` + `npm run deploy:gh-pages`：本地构建后用 git push -f 推 dist 到 gh-pages 分支，无需 GitHub Actions
