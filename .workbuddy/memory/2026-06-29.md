# 2026-06-29 工作记录

## 修复：腾讯蓝皮肤引入的浅色主题破坏问题

### 问题根因
上次制作腾讯蓝皮肤时将 AppSidebar.vue 中品牌图标/导航激活/用户头像从 `--color-success` 改为 `--color-primary`，导致浅色主题侧栏从绿色变成了紫蓝色（破坏了原有设计）。

### 修复方案（正确思路）
本应用的"主色调按钮"设计使用 `--color-success` 变量（绿色系），而非 `--color-primary`。

**恢复 AppSidebar.vue（3处）：**
- `.brand-icon`: primary-light/primary → success-light/success-text
- `.user-avatar` 渐变: primary/primary-text → success/success-text
- `.nav-item.active`: primary-light/primary-text → success-light/success-text

**在 style.css `[data-theme="tencent"]` 块中覆盖 success 系列为腾讯蓝：**
- `--color-success: #0052D9`
- `--color-success-light: #EDF1FF`
- `--color-success-lighter: #F2F3FF`
- `--color-success-text: #003CAB`

这样：浅色/暗色/ZURU 主题不受影响，腾讯蓝主题下所有用 success 变量的按钮、侧栏元素自动变蓝色。

### 经验教训（重要）
当引入新主题时，**不要改动组件 CSS 变量的引用**，应在新主题的 CSS 块中覆盖变量值。这样其他主题零影响。
