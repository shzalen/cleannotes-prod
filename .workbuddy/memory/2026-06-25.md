## Bug 修复：新建备忘录后编辑器自动回退

**现象**：点击「新建」进入编辑区域，若 800ms 内未输入任何内容，编辑器自动消失退回空状态。

**根因**：`createNewMemo()` 将 editTitle/editContent/editTags 全部置空时触发了 auto-save watcher → 800ms 后 flushAutoSave() 检测到标题内容均为空 → 执行"放弃新建"逻辑清除 isCreating。

**修复**：在 auto-save watcher 中加入哨兵条件——新建态且标题内容均为空时跳过调度，等用户真正开始输入后再启动定时器。

**文件**：`src/views/MemoView.vue`（第 88-93 行）
**构建**：零错误，✓ built in 3.98s

## Bug 修复：斜杠命令面板瞬间消失（cleannotes 主项目）

**现象**：在备忘录编辑器中输入 `/`，斜杠命令面板出现后立刻消失（~150ms）。

**根因**：`SlashCommand.ts` 的 `handleDOMEvents.blur` 处理器。面板打开后，Vue 在 `nextTick` 中将焦点移到面板内的 filter `<input>`，导致编辑器失焦 → blur 事件触发 → 150ms 后关闭面板。面板的筛选输入框和编辑器焦点天然互斥。

**修复**：
- `SlashCommand.ts` — 移除 `handleDOMEvents.blur`（替换为注释说明原因）
- `RichTextEditor.vue` — 复用现有 `mousedown` 委托逻辑：点击编辑器/斜杠面板外 → 关闭面板。与气泡菜单隐藏共用同一事件监听器

**文件**：`src/extensions/SlashCommand.ts`（移除 blur handler）、`src/components/RichTextEditor.vue`（合并 mousedown 逻辑）
**构建**：零错误，✓ built in 3.48s
