### 2.4 养成系统测试（烛）

#### 2.4.1 XP 计算规则验证

XP 计算公式（代码来源：`growth.ts:194-225`）：

| 来源 | XP | 条件 | 代码来源 |
|------|-----|------|----------|
| `complete`（基础完成） | +10 XP | 任何完成操作 | `calculateXp → complete: 10` |
| `priority`（高优先加成） | +5 XP | `task.priority === 'high'` | `calculateXp → priority: 5` |
| `night`（凌晨加成） | +3 XP | 完成时间在 0-6 点 | `calculateXp → night: 3` |
| `deadline`（截止日加成） | +5 XP | `task.dueDate` 等于今天 | `calculateXp → deadline: 5` |
| `streak`（连续天数加成） | N×2 XP | `streakDays > 0` | `calculateXp → streak: N*2` |
| `achievement`（成就解锁） | +20 XP | 首次解锁成就 | `checkAchievements → xp: 20` |

| 用例ID | 测试项 | 前置条件 | 操作步骤 | 预期结果 | 优先级 | 状态 |
|--------|--------|----------|----------|----------|--------|------|

| TC-GROW-002 | 高优先级加成（+5） | priority=high | 1. 完成一个高优先级任务 | 基础 10 + 优先级 5 = 15 XP | P0 | CHECK |

### 3.2 test-reports JSON 格式规范

```typescript
// 顶层结构
interface TestReport {
  version: string            // 版本号，如 "1.3.0"
  testDate: string           // 测试日期，如 "2026-06-30"
  testEnv: string            // 测试环境，如 "Windows + Chrome"
  testAccount: string        // 测试账号，如 "13800000000"
  summary: ReportSummary     // 汇总统计
  modules: Module[]          // 模块列表
  bugs: Bug[]               // 发现的问题
  findings: string[]        // 核心发现
  recommendations: string[] // 建议
}

// 汇总统计
interface ReportSummary {
  total: number              // 用例总数
  passed: number             // 通过数
  failed: number             // 失败数
  manualCheck: number        // 待手动验证数
  autoPassRate: string       // 自动化通过率，如 "98.5%"
  p0Total: number            // P0 用例总数
  p0Passed: number           // P0 通过数
  p1Total?: number           // P1 用例总数（可选）
  p1Passed?: number          // P1 通过数（可选）
  foundBugs: number          // 发现 Bug 数
}

// 模块（支持扁平用例和子模块两种组织结构）
interface Module {
  id: string                 // 模块 ID
  name: string              // 模块名称
  passRate: string          // 通过率，如 "100%"
  cases?: TestCase[]        // 扁平用例列表
  subModules?: SubModule[]  // 子模块列表
}

interface SubModule {
  name: string              // 子模块名称
  cases: TestCase[]         // 子模块用例如列表
}

interface TestCase {
  id: string                // 用例编号
  name: string              // 测试项名称
  priority: string          // 优先级：P0/P1/P2
  status: 'pass' | 'fail' | 'check'  // 测试结果
  method: string            // 测试方式：自动化/手动验证/代码审查
  note: string             // 备注（额外说明）
}

interface Bug {
  id: string                // Bug 编号，如 "BUG-001"
  severity: string          // 严重级别：P1/P2
  title: string             // Bug 标题
  description: string       // 描述
  reproduce?: string        // 复现步骤
  status: string            // 状态：open/fixed/closed
}
```

## 4. Bug 追踪规范

### 4.1 严重级别定义

| 级别 | 定义 | 示例 |
|------|------|------|
| **P0 - 阻断** | 核心功能完全不可用，阻塞发版 | 登录失败、数据丢失、页面白屏 |
| **P1 - 严重** | 重要功能异常，影响主要用户体验 | AI 输入框不可用、同步失败 |
| **P2 - 一般** | 次要功能异常或 UI 显示问题 | 按钮文本匹配不一致 |

### 4.2 复现步骤描述规范

每条 Bug 应包含：
1. **Bug ID**：自动编号（BUG-001, BUG-002...）
2. **标题**：简明描述问题
3. **描述**：问题的具体表现
4. **复现步骤**：最小化操作路径
5. **严重级别**：P0/P1/P2
6. **状态**：open（待修复）→ fixed（已修复待验证）→ closed（已验证关闭）

### 4.3 状态管理

```
┌──────┐    修复    ┌───────┐    验证通过    ┌───────┐
│ open │ ────────→ │ fixed │ ────────────→ │ closed │
└──────┘           └───────┘               └───────┘
     ↑                                         │
     └──────────── 验证不通过 ─────────────────┘
```

### 7.1 每次发版前必测项（P0 核心流程）

**认证**：
- [ ] 新用户注册登录完整流程（手机号→验证码→首页）
- [ ] 老用户直接登录
- [ ] 退出登录后回到登录页
- [ ] 未登录状态下 protected 路由重定向到登录页

**任务管理**：
- [ ] 创建任务（必填字段 + 可选字段）
- [ ] 任务三态流转（todo→in_progress→done）
- [ ] 已完成任务重新激活（弹出 ConfirmDialog 确认）
- [ ] 删除非已完成任务（ConfirmDialog danger 确认，移入回收站）
- [ ] 回收站恢复和永久删除

**备忘录**：
- [ ] 新建备忘 → 编辑内容 → 自动保存（800ms 防抖）
- [ ] 斜杠命令 `/` 触发面板
- [ ] Mermaid/思维导图插入和渲染
- [ ] 图片粘贴/拖入和 Lightbox 查看
- [ ] 删除备忘（danger 确认）

**待办**：
- [ ] 创建待办
- [ ] 转任务功能（openFromTodo 预填）
- [ ] 删除待办（danger 确认）

**养成系统**：
- [ ] 完成任务后 XP Toast 显示
- [ ] 等级提升 Toast 显示
- [ ] 成就解锁 Toast 显示
- [ ] 日状态刷新（首次 vitality）

**主题系统**：
- [ ] Light/Dark/Auto/ZURU 四套主题切换
- [ ] 主题持久化（刷新后保持）

**同步**：
- [ ] 登录后数据加载
- [ ] 离线操作无障碍
- [ ] 多设备数据合并验证

**周报**：
- [ ] 周报生成
- [ ] 统计数据分析结果正确性

**H5 移动端**：
- [ ] 移动端访问自动跳转 H5
- [ ] H5 任务创建/编辑
- [ ] 强制 PC 模式

**全局**：
- [ ] ConfirmDialog 确认/取消功能
- [ ] 所有删除操作有确认对话框
- [ ] 空状态显示正常
- [ ] 版本号在应用中正确显示
