## 1. 信息架构

### 1.1 页面层级结构

**PC端（10页）**

```
├── /login              登录页（公开路由）
├── /                   首页（仪表盘概览）
├── /tasks              任务列表
├── /todos              待办清单
├── /memos              备忘录（左右分栏）

├── /spirit             精灵养成（烛火状态）

```

**H5移动端（7页）**

```
├── /h5/tasks           任务列表
├── /h5/tasks/new       新建任务
├── /h5/tasks/:id       编辑任务
├── /h5/todos           待办清单
├── /h5/todos/new       新建待办
├── /h5/todos/:id       编辑待办
```

- **PC端**：左侧固定侧栏 + 右侧内容区，侧栏包含全部8个主要导航项

### 2.2 路由守卫逻辑

路由守卫 (`router.beforeEach`) 执行以下流程：

```
用户访问路由
    │
    ├──→ 目标路由为 H5 (/h5)?
    │       YES → 清除 force_pc 标记，恢复自动检测
    │
    ├──→ 目标路由为公开路由 (meta.public)?
    │       YES → 已认证?
    │               YES → 移动端? → 跳转 /h5/tasks
    │               YES → PC端?  → 跳转 /home
    │               NO  → 正常渲染公开页
    │
    ├──→ 目标路由为受保护路由
    │       未认证 → 跳转 /login
    │                (H5路由额外保存 h5_redirect 到 sessionStorage)
    │       已认证 → 设备自适应检查
    │                移动端访问PC路由 → 跳转 /h5/tasks (除非 force_pc)
    │                PC端访问H5路由  → 跳转 /home
    │
    └──→ 认证通过 + 设备匹配 → 正常渲染
         同时 switchUser(userId) 确保存储适配器正确
```

**关键规则**：
- `isMobileDevice()` = UA关键词匹配 **OR** `window.innerWidth <= 480`
- `isForcePC()` = localStorage 有 `cleannotes_force_pc=1` 标记
- 用户主动进入H5 → `clearForcePC()`，恢复自动检测
- 用户在设置页手动切换桌面版 → `setForcePC()`，阻止自动跳H5

## 3. 核心交互状态机

### 3.1 任务三态循环

**状态流转**（源码: `stores/task.ts:225-227`）

```
  ┌───────┐
  │  todo │ ──────→ ┌──────────────┐ ──────→ ┌──────┐
  │ (待办) │         │ in_progress  │         │ done │
  └───────┘         │  (进行中)     │         │(完成)│
      ↑              └───────┬──────┘         └───┬──┘
      │                      │                    │
      └──────────────────────┴────────────────────┘
                     done → todo (需确认弹窗!)
```

**状态转换触发**：

| 当前状态 | 下一状态 | 触发方式 | 是否需要确认 |
|---------|---------|---------|------------|
| todo | in_progress | `toggleStatus()` | 不需要 |
| in_progress | done | `toggleStatus()` | 不需要 |
| done | todo | `requestToggleStatus()` → 确认弹窗 | **需要确认** |

**已完成→待办确认弹窗流程**（源码: `stores/task.ts:230-264`）

```
用户点击 done 状态的任务
    │
    ├──→ requestToggleStatus(id)
    │       task.status === 'done' ?
    │       YES → 设置 reactivateConfirm:
    │              { visible=true, taskId, taskTitle }
    │              弹出 ConfirmDialog
    │              "确定要将此任务重新激活为待办吗？"
    │       NO  → 直接 toggleStatus()
    │
    ├──→ 用户确认 confirmReactivate()
    │       有 extraPatch? → updateTask(id, extraPatch)
    │       无 extraPatch? → toggleStatus(id)  // done→todo
    │       关闭弹窗
    │
    └──→ 用户取消 cancelReactivate()
            关闭弹窗，状态不变
```

**时间字段自动填充**（源码: `stores/task.ts:185-191`）：

| 状态变更 | 自动行为 |
|---------|---------|
| → in_progress | 自动填充 `inProgressAt` (当前UTC时间) |
| → done | 自动填充 `completedAt` (当前UTC时间) |
| done → todo | 清除完成时间相关字段(由toggleStatus驱动) |

**XP触发**（源码: `stores/task.ts:196-198`）：

- 仅在 `status === 'done'` 且 `wasDone === false` 时触发 `onTaskDoneCallback`
- 即：从未完成→完成 才触发XP计算，重新激活后再次完成也会触发

### 3.2 回收站操作流程

```
用户删除任务 (deleteTask)
    │
    ├──→ task.status === 'done'? → 拒绝删除（已完成不允许删除）
    │
    ├──→ 创建 DeletedTask: { ...task, deletedAt: now }
    │    移入 trash 数组
    │    从主列表移除
    │
    └──→ 回收站中的操作:
         ├── restoreTask(id)     → 恢复到主列表（状态不变）
         ├── permanentDelete(id) → 永久删除 + 记录 tombstone
         └── emptyTrash()        → 清空全部 + 批量 tombstone
```

**7天自动清除**（源码: `stores/task.ts:106-129`）：

- `TRASH_EXPIRE_DAYS = 7`
- 每次 `load()` 时执行 `purgeExpired()`
- 计算方式: `now - deletedAt >= 7 * 24h`
- 超期任务: 从trash移除 → 记录tombstone → 从Supabase两个表删除
- `expiringTrash` 计算属性: 剩余 ≤ 3天的任务（用于UI提醒）
- `getRemainingDays()`: 返回剩余天数(向上取整，最小0)

**tombstone机制**（源码: `services/hybrid.ts`）：

- tombstone 是永久删除标记，防止同步恢复已彻底删除的任务
- `addTombstones(ids)` → 记录到全局集合
- `getTombstones()` → 返回 Set，用于过滤同步数据
- `removeTombstones([id])` → 恢复任务时移除标记

### 3.3 养成系统状态机

**三态定义**（源码: `stores/growth.ts`）

```
  ┌───────────┐          ┌───────────┐          ┌───────────┐
  │ vitality  │ ←──────→ │ recovery  │ ←──────→ │ withered  │
  │ (活力态)  │          │ (复苏态)  │          │ (倦意态)  │
  └───────────┘          └───────────┘          └───────────┘
```

**状态转换条件**：

| 转换方向 | 条件 | 触发时机 |
|---------|------|---------|
| → vitality | 今日完成 ≥ 3 个任务 | `applyXp()` 中实时判定 |
| → recovery | 倦意态下完成 ≥ 1 个任务 | `applyXp()` 中实时判定 |
| → withered | 中断多天未活动 | `refreshDailyState()` 每日重置 |
| recovery → vitality | 连续2天完成（streakDays累加） | `refreshDailyState()` |
| vitality → recovery | 昨天有活动但未达到活力标准 | `refreshDailyState()` |

**每日状态重置流程**（源码: `stores/growth.ts:142-181`）

```
refreshDailyState() 在 load() 时执行:
    │
    ├──→ lastActiveDate 为空? → 首次使用，设为 vitality
    │
    ├──→ lastActiveDate === 今天? → 已刷新过，不操作
    │
    ├──→ lastActiveDate === 昨天? → 连续活动
    │       streakDays += 1
    │       maxStreakDays = max(max, streakDays)
    │       当前状态?
    │           withered → recovery (witheredDays=0)
    │           其他     → vitality
    │
    └──→ lastActiveDate 早于昨天? → 中断
            streakDays = 0
            gapDays = 计算间隔天数
            witheredDays = max(witheredDays + gapDays - 1, 1)
            dailyState = withered
            │
    └──→ 更新 lastActiveDate = today, persistState()
```

**XP计算规则**（源码: `stores/growth.ts:194-225`）

| 来源 | XP值 | 条件 |
|------|------|------|
| complete（基础完成） | +10 | 完成任何任务 |
| priority（高优先加成） | +5 | task.priority === 'high' |
| night（凌晨加成） | +3 | completedAt 在 0-6 点 |
| deadline（截止日完成） | +5 | dueDate === 今天 |
| streak（连续天数加成） | streakDays × 2 | streakDays > 0 |
| achievement（成就解锁） | +20 | 任何成就解锁时额外奖励 |

**升级公式**：Lv.N → Lv.N+1 需要 `N × 20` XP

**成就系统**（源码: `stores/growth.ts:47-68`）

| 分类 | 成就 | 条件 |
|------|------|------|
| milestone | 破土/扎根/深根/参天/林海 | totalDone >= 1/50/200/500/1000 |
| streak | 春风/夏雨/秋实/岁寒 | maxStreak >= 7/30/100/365 |
| special | 星光/时钟/巧手/满月 | nightDone>=10 / deadlineDone>=20 / dailyDone>=10 / dailyRate==100% |
| hidden | 枯木逢春 | 倦意态后连续3天完成 (recovery + streakDays>=3) |
| hidden | 从头再来 | 删除后创建（条件不公开） |
| hidden | 星河入梦 | 连续3天凌晨完成（条件不公开） |

## 4. 富文本编辑器交互

### 4.1 斜杠命令

**触发机制**（源码: `extensions/SlashCommand.ts:417-423`）

- 用户在段落开头（`$pos.parentOffset === 0`）输入 `/` 字符
- `handleTextInput` 检测 `/` + 段首位置 → 调用 `showPanel()`
- 纯 ProseMirror `handleTextInput` + 原生 DOM 面板，零依赖于 @tiptap/suggestion

**搜索筛选交互**：

- 面板打开后，`startTracking()` 通过 `requestAnimationFrame` 持续追踪光标位置
- 每帧读取光标到 `/` 之间的文本作为查询词 `_query`
- `getFilteredIndices(query)` 匹配 `label` 和 `type`（不区分大小写）
- 查询词变化 → `renderPanel()` 重渲染
- 无匹配时显示 "没有匹配的结果"

**键盘导航**（源码: `extensions/SlashCommand.ts:335-366`）

- 事件注册在 `document` 的 **CAPTURE 阶段**，确保在 ProseMirror 处理之前拦截
- 支持的按键：

| 按键 | 行为 |
|------|------|
| ↑ | `_selectedIdx` 循环上移，`updateHighlight()` + `scrollSelectedIntoView()` |
| ↓ | `_selectedIdx` 循环下移，同上 |
| Enter | 执行 `onSelect(indices[_selectedIdx])` |
| Escape | `closePanel()` + `view.focus()` 回到编辑器 |
| 其他按键 | 不拦截，正常传递到编辑器 |

- 所有导航键均 `preventDefault + stopImmediatePropagation`，阻止 ProseMirror 接收

**16种块类型菜单结构**（源码: `extensions/SlashCommand.ts:29-46`）

| 分类 | 类型 | 说明 |
|------|------|------|
| 基础块 | text, h1, h2, h3 | 段落 / 一级/二级/三级标题 |
| 列表 | bullet, ordered, taskList | 无序/有序/待办清单 |
| 高级块 | quote, callout, toggle, hr, codeBlock, table | 引用/提示/折叠/分割线/代码/表格 |
| 高级块 | mermaid, mindmap | Mermaid图表 / 思维导图 |
| 嵌入 | memoLink | `[[` 双括号链接备忘录 |

**面板定位**：

- `fixed` 定位，基于 `coordsAtPos()` 计算光标下方 6px
- 自动翻转：超出底部 → 显示在光标上方
- 自动收缩：超出右侧 → 向左偏移
- 最大宽度 260-320px，最大高度 340px

**鼠标交互**：

- `mouseenter` → CSS-only 高亮（仅修改 `_selectedIdx` + `updateHighlight()`，不重建DOM）
- 面板内 `mousedown` → 事件委托 `closest('[data-slash-idx]')` → `onSelect()`
- 面板外 `mousedown` (CAPTURE) → `closePanel()`
- 编辑器内 `handleClick` → 非 `.rte-slash-panel` 区域点击 → `closePanel()`

**选择执行** (`onSelect`)：

- 保存 editor/range/view 引用 → 删除 `/` + 查询词文本 → 应用块类型 → 关闭面板
- 顺序严格：先执行 `chain().deleteRange().toggleBlock()`，再 `closePanel()`

**编程式打开**：

- `openSlashPanel(editor)` → 在光标位置插入 `/` → `showPanel()`
- 由 `+` 快速插入按钮调用

### 4.2 浮动气泡菜单

**触发机制**（源码: `components/RichTextEditor.vue:136-158`）

- `document.addEventListener('mouseup', showBubble)` — 全局 mouseup 监听
- 条件检查：
  - 选中文字不为折叠（`!sel.isCollapsed`）
  - 选中范围在编辑器内（`rteWrapperRef.contains`）
  - 选区宽度 > 0
  - 当前不在表格内（`!editor.isActive('table')`）

**定位**：

- `fixed` 定位
- `top = rect.top - 44`（选区上方 44px）
- `left = rect.left + rect.width / 2`
- `transform: translateX(-50%)` 水平居中

**操作按钮**（6个功能区）：

| 区域 | 按钮 | 行为 |
|------|------|------|
| 文本格式 | B / I / S / U | toggleBold / toggleItalic / toggleStrike / toggleUnderline |
| 颜色 | 颜色圆圈 | 打开 10色颜色选择器（`TEXT_COLORS`） |
| 高亮 | 方框图标 | toggleHighlight（#fef08a 黄色标记） |
| 对齐 | 对齐图标 | 打开 4向对齐选择器（左/中/右/两端） |
| 转换 | 循环箭头 | 打开"转换为"面板（9种块类型） |
| 链接 | （无独立按钮） | `bubbleToggle('link')` → `window.prompt('输入链接地址:')` |

**隐藏机制**：

- 点击气泡外区域 (`mousedown`) → `hideBubble()`
- 编辑器滚动 → `hideBubble()`
- 进入表格 → `hideBubble()`（避免与表格控件冲突）
- 选中图片 → `hideBubble()`（切换为图片工具栏）

### 4.3 + 快速插入

**触发机制**（源码: `components/RichTextEditor.vue:952-968`）

## 5. 拖拽排序交互

**备忘录拖拽重排**（源码: `views/MemoView.vue:417-467`）

```
拖拽条件: canDrag = !searchQuery && !activeTag (无搜索/无标签过滤时才可拖拽)
    │
    ├──→ onDragStart(e, memo)
    │       设置 dragId, dragPinned
    │       effectAllowed = 'move'
    │
    ├──→ onDragOver(e, memo)
    │       检查: 同组(pinned/normal)才能拖拽
    │       计算中线: e.clientY > midY → dragOverAfter=true
    │       视觉反馈: drag-over / drag-over-after CSS类
    │
    ├──→ onDrop(e, memo)
    │       调用 store.reorderMemo(dragId, targetIndex)
    │
    └──→ onDragEnd → 清除所有拖拽状态
```

**stride=1000 设计原理**（源码: `stores/memo.ts:158-181`）

- `sortOrder` 使用 `i * 1000` 作为排序值
- 新建备忘录: `sortOrder = minOrder - 1000`（排在最前）
- 置顶/取消置顶: `sortOrder = minOrder - 1000`（目标组最前）
- 拖拽重排: 重新分配 `i * 1000`
- **好处**: 只需更新实际移动项的 sortOrder，而非全部项；1000间隔允许最多999次中间插入而不需要全量重排

### 8.5 完成庆祝卡片

**源码**: `composables/useCompletionCelebration.ts`

**触发条件**：

```
监听 [todayTotal, todayDone, loaded]
    │
    ├──→ loaded=false? → 不触发
    ├──→ total=0? → 不触发
    ├──→ done !== total? → 不触发
    ├──→ hasShownToday()? → 不触发（每日只显示一次）
    │
    └──→ 条件满足 → 600ms debounce
           防止同步波动导致误触发
           │
           └──→ 确认 done===total && 未展示 → visible=true + markShownToday()
```

- `hasShownToday()`: 检查 localStorage `cleannotes_${userId}_last_celebration === 今天日期`
- `markShownToday()`: 写入今天日期到 localStorage
- 每日只触发一次，隔天重置
