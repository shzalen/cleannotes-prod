三、发现的缺陷详情

BUG-01: H5Settings.vue h5Confirm 导入错误

P0 严重 已修复
影响：H5 设置页所有确认对话框静默失败 | 发现时间：2026-07-13 交互测试 | 模块：H5Settings.vue

问题描述：

H5Settings.vue 使用 const { h5Confirm } = useH5Dialog() 从 composable 返回值中解构 h5Confirm，但 useH5Dialog() 仅返回 { state, onConfirm, onCancel }，h5Confirm 是独立的模块级命名导出。解构结果为 undefined，导致 await h5Confirm(...) 抛出 TypeError，async onLogout() 静默失败——无确认对话框、无登出、无页面跳转。

根因分析：

// ❌ 错误写法（H5Settings.vue 原代码） import { useH5Dialog } from '@/composables/useH5Dialog' const { h5Confirm } = useH5Dialog() // h5Confirm = undefined // ✅ 正确写法（H5TaskEdit/H5TodoList/H5TodoEdit 中的模式） import { h5Confirm } from '@/composables/useH5Dialog' // 直接命名导入

修复方案：

修复内容
  • 将 import { useH5Dialog } + const { h5Confirm } = useH5Dialog() 改为 import { h5Confirm }（直接命名导入）
  • onLogout() 函数恢复原始实现：const confirmed = await h5Confirm('确定退出登录？', '退出登录'); if (!confirmed) return; auth.logout(); router.push('/login')
  • goPC() 函数恢复原始实现
  • 构建验证：npx vite build exit 0（24.18s）

useH5Dialog.ts 架构说明：

// 模块级导出（独立函数，直接 import 使用） export function h5Confirm(message: string, title?: string): Promise<boolean> export function h5Alert(message: string, title?: string): Promise<void> // composable 返回值（供 H5Layout.vue 绑定 Dialog 组件） export function useH5Dialog(): { state: Ref<DialogState>, onConfirm: () => void, onCancel: () => void }

四、需真机验证项

自动化测试局限性

agent-browser 的 accessibility tree 点击定位对 H5 移动端嵌套结构（div 内含 SVG 图标）的可靠性有限。以下两项在代码层面已确认修复，但交互层面需在真实移动设备上二次验证：

  • IT-47 H5 ConfirmDialog：h5Confirm 导入已修复为正确模式，确认对话框组件绑定逻辑无误，但自动化点击事件可能未正确传播至嵌套 div+SVG 元素。真机上触摸事件应能正常触发。
  • IT-48 H5 登出：onLogout() 代码逻辑已恢复为完整的 h5Confirm → auth.logout → router.push 流程，但自动化浏览器点击"退出登录"按钮不稳定。真机上应能正常工作。
