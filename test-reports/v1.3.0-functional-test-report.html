🐛 缺陷汇总（去重后 17 项）
高 DEF-01 AI taskId 模糊匹配导致误操作风险
模块: ai.ts
描述: taskId.startsWith(args.taskId) 用于 AI 工具调用的任务匹配。短 ID（如 "1"）会匹配多个任务，破坏性操作（update_task / delete_task）可能导致非预期任务被修改或删除。
影响: AI 误删/误改用户任务数据
改为精确匹配 args.taskId === taskId，模糊匹配仅在 list_task 等非破坏性操作中使用。
中 DEF-02 登出时数据 flush 顺序错误导致数据丢失
模块: AppSidebar.vue:120-127 + App.vue:100-120
描述: confirmLogout() 先调用 auth.logout() 清除 JWT 和 switchUser('') 清空 userId，然后 emit('logout') 才触发 handleLogout() 执行 flush。此时 token 已为 null、userId 已为空：
• memo pendingWrites 被 setMemoUserId('') 清空
• task upsert 的 user_id 为空，被 RLS 拒绝
• growth flush 因 currentUserId 为空直接 return
影响: 用户登出前 2 秒内编辑的备忘录、300ms 内修改的任务、2s 内获得的成长数据可能丢失
将 confirmLogout() 中的 auth.logout() 和 switchUser('') 移除，仅 emit('logout')，由 handleLogout() 统一处理（先 flush → 再清理 → 最后跳转）。
中 DEF-03 growthStorage 缺少 visibilitychange flush
模块: growthStorage.ts
描述: growthStorage 使用 2s 防抖上传，但没有注册 visibilitychange 监听器。task.ts 和 memoStorage.ts 都有 visibilitychange 处理，但 growthStorage 遗漏。当用户完成任务获得 XP 后立即关闭浏览器标签，2s 防抖 timer 可能未触发，导致 XP 数据丢失。
影响: 关闭标签页时最近 2 秒内获得的 XP 数据可能丢失
在 growthStorage.ts 中添加 document.addEventListener('visibilitychange', ...)，页面隐藏时调用 flushGrowthToCloud()。可利用 beforeunload 或 pagehide 事件作为额外保障。
中 DEF-04 cycleStatus 未来任务检查不一致
模块: TaskRightPanel.vue — cycleStatus() 行632
描述: cycleStatus() 使用 task.createdAt.slice(0,10) > today 检查未来任务，而同文件的 isFutureTask() 正确优先检查 task.startDate。模板中 !isFutureTask(task) && cycleStatus(task) 的短路求值提供了保护，但如果 cycleStatus 通过其他路径被调用可能绕过锁定。
影响: 潜在的未来任务状态被意外切换
将 cycleStatus() 中的检查替换为 if (isFutureTask(task)) return，复用统一逻辑。
中 DEF-05 周报 AI 摘要生成失败时静默跳过
模块: weeklyReport.ts — generateAiSummary()
描述: Phase 2 AI 摘要生成失败时返回 null（静默跳过），用户只看到"生成失败"占位文本，无重试按钮和具体错误信息。
影响: 用户无法得知失败原因，无法手动重试
添加重试按钮 / UI 反馈，显示具体失败原因（超时 / API 错误等），允许用户手动重新生成。
中 DEF-06 deleteMemoById 静默吞错，删除操作无重试
模块: memoStorage.ts — deleteMemoById()
描述: 删除操作使用 .catch(() => {}) 静默吞错，未纳入 pendingWrites 重试队列。网络异常时删除可能丢失，用户以为已删除但服务器仍保留。
影响: 网络不稳定时删除操作可能不生效
将删除操作纳入重试队列，或至少记录失败日志供用户感知（如 toast 提示"删除失败，已加入重试队列"）。
低 DEF-07 TaskEditModal 保存按钮缺少 disabled 状态
空标题时按钮可点击但无反应，与 TodoEditModal 不一致（后者有 :disabled）。
给 btn-save 添加 :disabled="!title.trim()"。
低 DEF-08 getHeatmapData 重复实现
task.ts 导出 getHeatmapData()，但 TaskHeatmap.vue 在组件内重新实现相同逻辑。统计口径变化时需同步修改两处。
移除 TaskHeatmap.vue 内联计算，改用 store.getHeatmapData(view)。
低 DEF-09 SettingsView 缺少数据导出/导入功能
设置页无 JSON 导出/导入入口。如计划后续版本实现，当前标注"未实现"即可。
确认是否为 v1.3.0 计划功能；若是则补充实现，若否则标记为 roadmap。
低 DEF-10 dailyMessage 计算属性每次重算随机
computed(() => { ... Math.random() ... }) 在任何响应式更新触发重算时都会随机切换寄语，导致 UI 不稳定。
在 refreshDailyState() 中一次性决定并存储到 state（如 state.dailyMessageIndex）。
低 DEF-11 SpiritIllustration SVG 渐变 ID 冲突风险
SVG 渐变使用固定 ID（haloGrad 等），多实例并存时可能冲突。当前应用只有 SpiritView + SpiritWidget 同时存在，影响有限。
使用 :id="'haloGrad-' + size" 动态 ID。
低 DEF-12 SlashCommand / DoubleBracketLinker 模块级单例状态
使用模块级变量（_editor / _view / _range 等），多编辑器实例并存时状态可能冲突。当前应用同时只有一个编辑器实例，影响有限。
改为编辑器实例级存储（如 editor.storage 或 PluginKey）。
低 DEF-13 flushAutoSave 乐观写入 ID 校正缺失
创建新备忘录时乐观写入（先创建后等服务器确认），若服务器失败，本地 ID 与服务器 ID 不一致。
添加服务器响应后的 ID 校正逻辑，或创建失败时回滚本地状态并提示用户。
低 DEF-14 closeCrossTabSync() 定义但从未调用
登出时未显式关闭 BroadcastChannel，依赖页面重载隐式清理。功能上无影响，但属于不完整的清理模式。
在 handleLogout() 中 broadcastChange('logout') 之后调用 closeCrossTabSync()。
低 DEF-15 clearOnTaskDone() 定义但从未调用
onTaskDoneCallback 在登出时未清除，依赖页面重载隐式清理。回调可能持有 growth store 引用。
在 handleLogout() 中调用 clearOnTaskDone()。
低 DEF-16 healthCheckTimer 未在登出时停止
stopAllSync() 是空函数，未调用 stopHealthCheck()。15s setInterval 在登出后继续运行直到页面重载。纯在线模式下 healthCheck 可能未实际启动。
在 stopAllSync() 中调用 stopHealthCheck()。
提示 DEF-17 5 个 Store 缺少 loadPromise 防竞态保护
仅 task.ts 有 loadPromise 缓存。memo/growth/ai/todo/weeklyReport 仅用 loaded.value flag，并发调用 load() 可能发起重复请求。实际触发概率低（App.vue Promise.all 只调用一次；跨标签同步触发 load(true) 时才可能竞态）。
可为其他 store 添加类似 task.ts 的 loadPromise 机制，提升健壮性。
